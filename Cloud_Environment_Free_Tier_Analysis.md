# 雲端環境免費方案分析報告

**文件版本:** 1.0
**日期:** 2025-12-04
**目標:** 確認用於建置與部署 `Taiwan-Tongues-ASR-CE` 模型的雲端環境，其免費方案的可行性。

---

## 結論摘要

1.  **建置 Docker Image**: **是**，我們可以透過 **GitHub Actions** 或 **Google Cloud Shell** 等服務**免費**完成此任務。
2.  **部署模型服務 (可呼叫狀態)**:
    *   **本機開發**: **是**，使用 Docker Desktop 在我們的專案規模下是免費的。
    *   **Staging 階段 (Railway)**: **部分免費**。核心的 ASR 模型因需要 GPU 而無法免費部署，需採替代方案。
    *   **生產階段 (GCP)**: **否**。執行 AI 模型所需的 **GPU 資源是付費服務**，這將是專案的必要成本。

---

## 1. Docker Image 建置環境 (Build Environment)

**第一性原理**: 此階段的任務是找到一個能執行 `docker build` 指令的環境。

| 方案 | 工具 | 執行方式 | 是否免費？ | 說明 |
| :--- | :--- | :--- | :--- | :--- |
| **A. 自動化 (建議)** | **GitHub Actions** | 在 Git 倉庫中設定 CI/CD 工作流程，當程式碼推送時，自動在雲端執行 `docker build` 並將 Image 推送到 **GitHub Container Registry (GHCR)**。 | **是** | GitHub 為公開專案提供免費的執行時間，私有專案每月也有 2,000 分鐘的免費額度，這對於 MVP 階段的建置完全足夠。 |
| **B. 手動** | **Google Cloud Shell** | 開發者手動登入 GCP 的線上終端機，Clone 專案後執行 `docker build`。 | **是** | Cloud Shell 本身免費，每週有 50 小時使用額度。但需注意其只有 5GB 的永久性磁碟儲存空間，若模型 Image 過大可能會建置失敗。 |

**建置環境結論**: 我們**可以免費完成**模型容器化與 Image 建置的任務，推薦使用 **GitHub Actions** 實現自動化，以符合 DevOps 最佳實踐。

---

## 2. 模型部署環境 (Deployment Environment - 可呼叫狀態)

**第一性原理**: 此階段的任務是找到一個能長時間運行 Docker 容器，並提供 GPU 運算資源（針對 ASR 模型）的環境。

| 部署階段 | 部署目標 | 工具/平台 | 是否免費？ | 關鍵說明 |
| :--- | :--- | :--- | :--- | :--- |
| **階段一 (本機)** | 在開發者電腦上運行 | **Docker Desktop** | **是** | 根據 Docker 的授權條款，對於小型企業（少於 250 名員工且年營收低於 1000 萬美元）、個人使用及開源專案是免費的。這完全涵蓋我們的 MVP 開發階段。 |
| **階段二 (Staging)** | 部署到雲端服務驗證 | **Render** | **否 (對於 ASR/LLM 模型)** | Render 的免費方案**不包含 GPU 資源**，且僅提供 0.1 CPU 和 256 MB RAM 的實例，閒置 15 分鐘會休眠。這表示 `Taiwan-Tongues-ASR-CE` 模型無法運行，而即使是 CPU-based 的 LLM 服務也可能因資源限制和休眠問題影響性能。此階段的 **ASR/LLM 服務需採用替代方案**（如暫時串接第三方 ASR API），但其他非資源密集型服務（如 FastAPI 閘道）可以在其免費方案下運行。 |
| **階段三 (生產)** | 部署到最終生產環境 | **GCP (GKE)** | **否** | **任何主流雲端平台（包含 GCP）的 GPU 資源都不是免費的**。雖然 GCP 提供新使用者 300 美元的免費額度，可以用於體驗 GKE 上的 GPU 實例，但這僅為一次性試用。一旦進入長期穩定運行階段，**GPU 的使用將會產生費用**。這部分的預算需要被納入考量。 |
