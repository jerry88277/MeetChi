> Checkpoint 目標：**讓「沒有知識背景、從未使用過」的 User，能單靠 App 內建的功能導覽，自行理解系統的使用方式（自學成功）。**

## 0. 角色策略決議

**結論：在原本三角色之外，新增第四角色「🧊 冷啟動零基礎使用者（Cold-start）」。**

理由：
- 原「🟢 新手」角色（行政助理）**仍預設了情境**——被主管要求試用、有動機、有耐心、看得懂「轉錄／摘要」等詞、願意讀說明。
- 本 checkpoint 要驗證的是更嚴苛的下限：**完全沒人教、第一次打開、會反射性關閉彈窗、不讀長文、用詞零基礎**的使用者，能否「只靠功能導覽」把系統學會。
- 這是一條跨功能的「自學專線」，不適合用舊三角色重跑，需專屬視角。

### 冷啟動角色定義
| 屬性 | 設定 |
|------|------|
| 背景 | 從沒用過任何 AI 轉錄/會議工具；不知道「轉錄＝語音轉文字」「摘要」「ASR」「RAG」 |
| 行為 | 反射性關閉彈窗/導覽；不讀超過 2 行的文字；靠「點點看」學習 |
| 心態 | 沒有外部說明文件、沒有人帶；卡住就放棄或亂點 |
| 成功標準 | 不需任何外部協助，能完成「上傳 → 看懂結果 → 知道還能做什麼」 |

## 2. 冷啟動角色 操作軌跡稽核（邊做邊碎念）

1. **登入頁** — 看到「MeetChi」+「把每場討論整理成下一步」
   - 😰「這到底是什麼？把什麼整理？是行事曆嗎？」→ 無一句話價值說明
2. **登入後進 dashboard，導覽自動跳出** ✅
   - 反射動作：右上角 X 或「跳過導覽」直接關掉（cold user 高機率）
   - ❌ 一旦關掉 → 旗標寫入 → **永遠不再自動出現**；零會議使用者沒有任何二次提示說「要不要看導覽」
   - 即使看完：步驟文字含「轉錄／摘要／決策／待辦／風險／ChiMemo」等詞，零基礎者**不一定懂「轉錄」是語音轉文字**
3. **看完導覽，面對零會議空畫面**
   - ✅ CTA「上傳第一場會議」清楚
   - 😰 但「我手上沒有錄音檔可測」「我不知道上傳完會長怎樣」→ **沒有範例/示範會議可先看成果**
4. **點上傳** — 跳出語言（國/國台英）、模板選擇、情境描述、機密標記
   - 😰「模板要選哪個？」導覽完全沒提到模板
   - 😰「情境描述要寫什麼？」「機密是什麼意思？」→ 選項無引導
5. **處理中** — 有心跳秒數 + toast ✅（前次稽核已修）
6. **完成，點開會議卡片看結果**（這是價值兌現的關鍵畫面）
   - ❌ DetailView **第一次打開沒有任何導覽**：摘要/決策/待辦分頁、講者分離、逐字稿、重新生成——全靠自己摸
   - 導覽**結構性到不了這裡**（新使用者沒有會議，spotlight 無對象）
7. **想找「跨會議查詢」** — sidebar 看到「ChiMemo」
   - 😰 若導覽被跳過，「ChiMemo」是無意義的品牌字，nav 上沒有副標
8. **卡住想找說明** — 全 App **沒有 ? 說明鈕／說明中心／教學連結**，只有「回報問題」

## 3. 發現問題清單（依 checkpoint 目標校準嚴重度）

> 嚴重度定義（本稽核專屬）：P0＝直接讓「自學」失敗；P1＝顯著阻礙自學；P2＝體驗打磨

| # | 問題 | 維度 | 嚴重度 | 說明 / 建議 |
|---|------|------|--------|-------------|
| **CS-1** | 導覽可被反射性跳過，且零會議使用者**無二次提示**、旗標永久 | 流程可用性 | **P0** | 違反 checkpoint 核心前提。建議：跳過時不立刻寫永久旗標；零會議首頁顯眼處放「▶ 觀看 1 分鐘導覽」常駐卡，直到用戶完成第一場會議 |
| **CS-2** | 導覽**到不了「會議詳情」這個價值兌現畫面**，且無範例/示範會議 | 流程可用性 | **P0** | 冷啟動者無法在投入自己檔案+等待前理解「會得到什麼」。建議：內建一筆唯讀「範例會議」(demo)，讓用戶先點開看摘要/決策/待辦長相；或 DetailView 首次開啟加 coachmark |
| **CS-3** | 登入頁/首頁無「這是什麼」一句話價值說明 | 信任/安全感 | P1 | 加一句「會議錄音 → AI 自動轉文字並整理出摘要、決策、待辦」 |
| **CS-4** | 導覽用詞預設詞彙（轉錄/摘要/RAG）對零基礎者過難 | 一致性 | P1 | 首步補白話：「把開會錄音變成文字，再自動幫你整理重點」 |
| **CS-5** | 上傳表單的模板/情境/機密選項**無引導**，導覽也沒提模板 | 流程可用性 | P1 | 模板預設「通用」+ tooltip「不確定就用通用」；情境/機密加一句說明 |
| **CS-6** | DetailView 首次開啟無任何說明（分頁/講者/重生摘要） | 流程可用性 | P1 | 首次開啟用 coachmark 點出「摘要分頁」「逐字稿」「重新生成」 |
| **CS-7** | 全 App 無常駐說明入口（? / 說明中心 / 教學） | 信任/安全感 | P1 | sidebar 加「說明/教學」入口，含重播導覽 + FAQ |
| **CS-8** | 導覽未涵蓋：模板、設定、錄音、語言、機密旗標 | 一致性 | P2 | 擴充導覽步驟或分「進階導覽」 |
| **CS-9** | 「ChiMemo」nav 標籤是品牌字，跳過導覽就無從理解 | 一致性 | P2 | nav 加副標「跨會議 AI 搜尋」或 hover tooltip |
| **CS-10** | 導覽只在 dashboard 路由觸發；落在其他頁不出現 | 邊界/錯誤處理 | P2 | 首次登入強制導向 dashboard 再觸發 |
| **CS-11** | 「即時錄音」標「開發中」仍是可點按鈕，冷啟動者會誤點困惑 | 邊界/錯誤處理 | P2 | disabled 樣式 + 明確「即將推出」 |
| **CS-12** | 導覽 spotlight 目標不存在時靜默置中，無 fallback 文字 | 非功能面 | P2 | 目標缺失時顯示「此功能在 XX 區」 |

## 4. 現況 vs 目標 — Gap 矩陣

| 自學旅程關卡 | 目標（零基礎可自學） | 現況 | Gap |
|--------------|----------------------|------|-----|
| 進場理解「這是什麼」 | 一眼看懂用途 | 只有 slogan | ⚠️ CS-3 |
| 首次導覽觸發 | 一定看得到、不易誤關失 | 自動開但易跳過、無二次提示 | ❌ CS-1 |
| 看懂「會得到什麼」 | 上傳前能看成果範例 | 無 demo、導覽到不了詳情 | ❌ CS-2 |
| 完成第一個動作（上傳） | 引導完成、選項有預設 | CTA 清楚，但表單選項無引導 | ⚠️ CS-5 |
| 看懂結果（詳情頁） | 首次開啟有說明 | 完全無 | ⚠️ CS-6 |
| 探索進階（ChiMemo/模板） | 名稱可理解、有引導 | 品牌字、導覽未涵蓋模板 | ⚠️ CS-8/9 |
| 卡住求助 | 有常駐說明入口 | 無（只有回報問題） | ⚠️ CS-7 |

**達成度評估**：核心「上傳」路徑 ✅ 已被導覽覆蓋；但「**不被跳過**」「**理解成果**」「**看懂詳情頁**」三個環節未達標 → 對「反射性跳過導覽」的冷啟動者，**自學會在第 2-3 關斷裂**。

## 7. 結論

```
冷啟動 Onboarding 稽核：2 P0 + 5 P1 + 5 P2 = 12 issues

現況：已有自製 5 步功能導覽 + 良好零會議 CTA + 豐富 ChiMemo 歡迎訊息，
      核心「上傳」路徑可被自學覆蓋。

未達標的關鍵斷點：
  1. 導覽易被反射性跳過且無二次提示（CS-1, P0）
  2. 看不到「成果長怎樣」、導覽到不了詳情頁、無範例會議（CS-2, P0）

最小可行改善（約 1-2 天）：
  - 零會議首頁常駐「觀看導覽」入口 + 跳過不寫死旗標（CS-1）
  - 內建一筆唯讀範例會議供先行探索（CS-2）
  這兩項可讓「反射性跳過導覽」的冷啟動者仍能自學成功，直接命中 checkpoint。

Onboarding 成熟度：β−
  - 有導覽骨架，但「防跳過 / 成果預覽 / 詳情頁引導」三個自學關鍵未補齊。
```
