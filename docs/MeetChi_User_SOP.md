## 3. 上傳會議錄音

### 3.1 支援格式
- 音訊：`.mp3`, `.wav`, `.m4a`, `.ogg`, `.webm`
- 影片：`.mp4`, `.mov`（系統自動提取音軌）
- 檔案大小上限：500MB

### 3.2 上傳步驟

```
操作流程：
  主畫面右下角「+」按鈕（或拖曳檔案至頁面）
    → 選擇音訊/影片檔案
      → 檔案加入上傳佇列（畫面底部顯示上傳列）
        → 上傳中：顯示進度百分比
          → 上傳完成：自動進入 AI 處理階段
            → AI 處理完成：會議卡片狀態變為「已完成」✓
```

### 3.3 上傳後的處理流程

| 階段 | 耗時（估計） | 說明 |
|------|-------------|------|
| 上傳 | 依檔案大小/網速 | 檔案傳輸至雲端 |
| 音源前處理 | ~30 秒 | 降噪、正規化 |
| 語音轉錄 (ASR) | 約 1/5 音檔時長 | GPU 加速轉錄 |
| 說話者辨識 | ~1-2 分鐘 | 區分不同說話者 |
| AI 摘要生成 | ~30 秒 | LLM 產生摘要與待辦 |

### 3.4 注意事項
- ⚠️ 上傳期間請勿關閉瀏覽器
- ⚠️ 若上傳失敗，底部佇列會顯示「重試」按鈕，點擊即可重新上傳
- ✅ 上傳完成後可離開頁面，AI 處理會在背景進行

### 5.3 提問技巧（重要❗）

#### ✅ 好的提問方式

| 提問 | 為什麼好 |
|------|---------|
| 「彙整最近所有提到 AI 投資 ROI 的討論」 | 聚焦主題關鍵字 |
| 「各場會議對 RAG 架構的看法有什麼共識或分歧？」 | 比較分析型 |
| 「客服流程改善在哪幾場會議被提到？」 | 追蹤特定議題 |
| 「鴻才討論的主要議題是什麼？」 | 用會議名稱 + 詢問內容 |

#### ❌ 避免的提問方式

| 提問 | 為什麼不好 | 改善建議 |
|------|-----------|---------|
| 「上週的會議」 | 太模糊，無關鍵字 | 指定主題：「上週討論的預算議題」 |
| 「那個檔案」 | 無法辨識 | 用會議名稱或主題描述 |
| 「全部」 | 範圍太廣 | 聚焦單一主題 |

### 5.4 回答結構

```
ChiMemo 回答格式：
  ┌───────────────────────────────────────┐
  │ 回答內容（彙整自多場會議）             │
  │                                       │
  │ 📎 參考來源：                          │
  │   [1] 會議名稱 A — 相關段落摘錄        │
  │   [2] 會議名稱 B — 相關段落摘錄        │
  └───────────────────────────────────────┘
```

- 點擊參考來源可跳轉至原始會議的對應段落

1. 進入會議詳情頁
2. 點擊標題旁的 ✏️ 鉛筆圖示
3. 輸入新名稱 → 按 Enter 或點擊確認

## 11. 常見問題 (FAQ)

### Q1：上傳後一直顯示「AI 處理中」怎麼辦？
**A**：AI 處理時間約為音檔時長的 1/5。一小時的錄音約需 12 分鐘。若超過 30 分鐘仍未完成，請透過問題回報功能通知管理員。

### Q2：ChiMemo 找不到我的會議內容？
**A**：
1. 確認該會議已完成 AI 處理（卡片顯示 ✓）
2. 用**主題關鍵字**提問，避免只用檔案名稱
3. 若用會議名稱提問，請加上「討論什麼」或「主要議題」等描述

### Q3：逐字稿的說話者標記不正確？
**A**：目前系統自動辨識說話者，準確度約 85-90%。若會議人數較多或環境噪音較大，可能影響辨識準確度。

### Q4：可以同時上傳多個檔案嗎？
**A**：可以。系統會自動排隊，依序處理。

### Q5：會議資料的安全性？
**A**：
- 所有資料存儲於奇美企業 GCP 專案中
- 傳輸使用 HTTPS 加密
- 頁面加上浮水印（顯示您的帳號）
- 僅授權使用者可存取

### Q6：支援哪些語言的轉錄？
**A**：目前支援：
- ✅ 中文（普通話）
- ✅ 英文
- ✅ 中英混合
- 🔜 台語（規劃中）
