### (2) ASR 低音量（人聽得到、電腦分析不出）調查
**現況 code 盤點（物理證據）**：
- `app/vad.py`：Silero VAD + **硬性 RMS 下限**：
  - `if is_speech and rms < 0.001: is_speech = False`（Silero 判為語音但太小聲 → 丟棄）
  - Energy fallback：`is_speech = rms > 0.005`
  - flush 階段 `SILENT_RMS_THRESHOLD = 0.0001` 丟棄近乎靜音段
- **全 pipeline 無任何響度正規化 / 增益**（grep 無 loudnorm/dynaudnorm/apply_gain/normalize）
- 空結果處置：`tasks.py` `[Offline ASR] Empty result` → 保留/略過；`empty_transcript` → `skipped`，
  但 meeting 仍標 COMPLETED，**前端無「未偵測到語音/音量過低」提示**（產品缺口）

**結論：確實存在「人聽得到但電腦分析不出」的情況**，且本系統目前落在此風險區：
1. **低音量（low amplitude, SNR 尚可）** → 應可救：正規化/增益後 ASR 可辨識。本系統因硬 RMS 門檻會直接丟棄。
2. **低 SNR（噪音淹沒語音）** → 純放大無效（噪音同步放大），需降噪/beamforming。
3. **削波失真（clipping）** → 放大更糟，需 declip 或重錄。

**解決方案（依 CP 值排序，來自多輪網路調查）**：
- **A. 前處理正規化（首選，低風險高回報）**：ASR 前對整檔做響度正規化
  - `ffmpeg -af "dynaudnorm"`（低音量/忽大忽小語音最有效）或 EBU R128 `loudnorm=I=-16:TP=-1.5:LRA=11`
  - 進階可 `dynaudnorm` 後接 `loudnorm` 二段式
- **B. RMS/LUFS 前置量測 + 條件式增益**：計算 dBFS，`< -35 dBFS` 才 boost；避免對正常音量重複處理
- **C. 放寬 VAD 門檻**：Silero `threshold` 調降（0.5→0.4）、移除/降低 `rms < 0.001` 硬丟棄（改為先正規化再判斷）
- **D. 產品面**：ASR 空結果時給 meeting 明確狀態（如「未偵測到語音」+ `failure_reason`），
  讓使用者區分「靜音」與「處理失敗」，並提示可重新上傳/放大
- **E. 邊界情況**：clipping/低 SNR 用純增益無效 —— 需 declip、降噪（RNNoise/DeepFilterNet）或提示重錄

**判斷指標（區分可救 vs 不可救）**：RMS dBFS（低音量）、spectral flatness（噪音 vs 語音）、
clipping 偵測（峰值連續觸頂比例）。true silence（頻譜平坦無結構）放大無意義。

1. 取乾淨語音（ASCEND/TALCS）→ 2. 依目標 SNR（20/10/0/-5 dB）以 RMS 對齊疊加 MUSAN 噪音
3. 低音量模擬：對乾淨語音施 -10/-20 dB 增益衰減（對應本專案「小聲但聽得到」情境）
4. 跑 ASR 量 WER/CER → 5. 對照「有/無前處理正規化（dynaudnorm/loudnorm）」比較改善幅度
- 關鍵：固定 random seed；torchaudio 無內建 add_noise，用 RMS 公式或 speechbrain/audiomentations
- 建議測試矩陣：SNR {20,10,0,-5} × gain {0,-10,-20} dB，驗證前處理對低音量/低 SNR 的邊界效果

### 對本專案（MeetChi）建議
- **最貼近情境**：TALCS（真實教學噪音 + 中英混）+ MUSAN 疊加做低音量壓測，驗證前一輪調查的
  方案 A（dynaudnorm/loudnorm 前處理）與方案 C（放寬 VAD `rms<0.001` 硬門檻）之改善。
- ASCEND 授權最寬鬆（CC-BY-SA），適合快速起步與 CI 迴歸小樣本。

### B. 空轉錄音檔診斷：bc41c457 / c2c92ef2（關鍵）

| **max_volume / Peak** | **-80.8 dBFS** | **-80.8 dBFS** | 最大峰值≈靜音底 |

已輸出 +80dB 放大 wav 供人工聆聽佐證：`~/asr_stresstest/prod_diag/{bc41c457,c2c92ef2}_amp80dB.wav`。

### C. 產品面待辦（承前一輪「方案 D」）
- 空／靜音音檔應於**上傳或轉錄前**偵測（如 mean_volume < -60dBFS 或全域 RMS < 門檻），
