# DevLog 2026-06-30: 上傳→轉錄 模組 UX 優化（批次 1）

## 背景

依 2026-06-30 三模組四角色稽核（`docs/audits/2026-06-30-module-upload-transcription-4persona-audit.md`），

③建立 Master TODO ④移除 Dashboard 綠標不實安全宣稱。本次執行批次 1。

## 執行的變更

### 變更 1: 上傳前設定視窗（U-A2 / T-A1，P0）✅
- **問題**：`DashboardView` 收了 `availableTemplates / uploadContext / uploadConfidential` 等 props 卻從未渲染，
  上傳永遠用 `general` / 空情境 / 非機密 → 模板管理的價值在主流程被埋沒。
- **修法**：
  - 新增 `components/UploadSettingsModal.tsx`：模板下拉（系統+自訂）、語言、情境、機密開關，支援 Esc / 背景關閉。
  - `app/dashboard/page.tsx`：選檔後不再直接上傳，改 `setPendingUploadFiles(files)` 開視窗；新增 `confirmUpload(settings)`
    執行檢查後 `enqueueFiles(...)`，並把設定回存為下次預設。

### 變更 2: transcribed 詳情頁不再空白（U-A1 / U-B5，P0）✅
- **問題**：卡片顯示「📄 逐字稿已可查看」，點進去 `DetailView` 因全部區段 gate 在 `isCompleted`，body 一片空白。
- **修法**：`DetailView.tsx` 新增 `isTranscribed`；逐字稿區段 gating 改 `(isCompleted || isTranscribed)`；
  新增「逐字稿已可查看／摘要生成中」橫幅 + 「立即生成摘要」；transcribed 時逐字稿預設展開。音檔播放器本就依 `audioUrl`
  且抓取僅依 `meeting.id`，故 transcribed 亦可播放。

### 變更 3: 機密會議「補真防護」（U-A3 / U-E1 / U-A4 / U-C6，P0）✅
- **問題**：`<SecurityWrapper>` 未傳參數 → 防護全預設關閉；🔒 標記與 tooltip 卻宣稱有保護（安全劇場）。
  且舊 MutationObserver 全域 subtree 監看易被瀏覽器擴充誤判、中止工作階段。
- **修法**：重寫 `components/SecurityWrapper.tsx`——
  - 攔截 copy / cut / contextmenu / selectstart / dragstart + Ctrl/⌘ + P/S/C/X/A；PrintScreen 盡力清剪貼簿並提示。
  - wrapper 套 `user-select: none`；浮水印改用實際登入者 email、可見度 0.04→0.1。
  - MutationObserver 只觀察浮水印節點屬性 + 父層 childList（不再全域 subtree）→ 降誤判。
  - `DetailView` 呼叫改為 `<SecurityWrapper isConfidential userIdentifier={userUpn}>`。

### 變更 4: 移除 Dashboard 不實安全宣稱（#4 / U-E2，P1）✅
- `DashboardView.tsx` 綠標 tooltip 由「不會將機密對話外洩至任何第三方雲端」
  改為「儲存於公司專屬 GCP 環境，透過存取控制限定授權人員」（與實際 GCS/Gemini 架構相符）。

### 變更 5: 上傳邊界與錯誤處理（U-C1 / U-C3 / U-C5，P1）✅
- `confirmUpload`：檔案 >2GB 或非 audio/video MIME → 個別 toast 略過，不阻斷其他合格檔案。
- `hooks/useUploadQueue.ts`：轉錄任務觸發失敗（原僅 console.error）改為 `toast.error` 並指引「重新從頭轉錄」。

### 變更 6: pending 詳情中文化（U-D2，P1）✅
- `DetailView.tsx` pending 區塊 `Status: QUEUED` → 「目前狀態：排隊中」。

## 執行的變更

### U-B2 上傳進行中可取消 ✅
- `lib/api.ts`：`uploadToGcs` / `resumableUpload` / `chunkedUpload` 新增 `signal?: AbortSignal` 參數，
  XHR 版透過 `signal.addEventListener('abort', () => xhr.abort())`，fetch 版直接傳 `signal` 並在迴圈檢查 `signal.aborted`。
- `hooks/useUploadQueue.ts`：新增 `abortControllersRef`（每 task 一個 AbortController），`executeUpload` 建立並傳入 signal；
  catch 內以 `controller.signal.aborted` 判定取消 → 清除孤兒 meeting + 靜默移除 task（不顯示錯誤）；新增 `cancelTask()`。
- `components/UploadTray.tsx`：`uploading` 狀態顯示常駐取消鈕（X），`queued` 移除鈕改為取消。
- `app/dashboard/page.tsx`：`<UploadTray onCancel={uploadQueue.cancelTask} />`。

### U-E3 刪除語意矛盾統一 ✅
- 後端 `meetings.py` 確認單筆刪除亦為 soft-delete（保留 30 天）。
- `page.tsx` 單筆刪除對話框文案由「且無法復原」改為「資料保留 30 天供 IT 還原」，與批次刪除一致。

### U-A5 / U-A6 假進度 / ETA ✅
- `MeetingCard.tsx`：移除以 stage 對應寫死百分比（15/50/65/85%）的假進度條，改為 `animate-indeterminate`
  不宣稱具體百分比的活動指示；移除未用的 `stageProgress()`。ETA 文字加「（預估）」+ tooltip 說明依歷史平均推估。
- `tailwind.config.ts`：新增 `indeterminate` keyframe/animation。

### U-C6 SecurityWrapper 誤判放寬 ✅（batch 1 已完成，本次補型別修正）
